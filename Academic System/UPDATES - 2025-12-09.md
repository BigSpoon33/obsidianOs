### Issue 2: Calendar View Parsing Error âœ…

**Problem:** Dataview query failed: `WHERE categories contains [[Coursework]]`

**Error Message:** "PARSING FAILED - Expected one of..."

**Fix:** Changed to simpler syntax: `WHERE categories = [[Coursework]]`

**Why It Works:**
- `=` operator works for single link values
- `contains` requires: `categories.contains(link("Coursework"))`
- Your solution was correct!

**File Modified:** `Examples/Academic Dashboard.md`

---

### Issue 3: Dynamic Heading Placeholders âœ…

**Problem:** `{{course-code}}` and `{{course-name}}` displayed as literal text

**Root Cause:** These were placeholder text, not actual template variables

**Fix:** Changed to Meta-bind VIEW syntax:
```markdown
# ğŸ“š `VIEW[{course-code}]` - `VIEW[{course-name}]`
```

**Result:** Heading updates dynamically when you change properties!

**Added:** Inline inputs for Course Code and Course Name in the callout

**File Modified:** `Templates/Class Template.md`

---

### Issue 4: Button Auto-Linking âœ…

**Problem:** Buttons created files but `class: [[]]` was empty

**Root Cause:** Meta-bind `{{course-code}}` syntax doesn't work - buttons can't access parent note's frontmatter

**Solution:** Created Smart Coursework Template with intelligent detection:

1. **Detects parent class** from recently opened files
2. **Finds** the most recent note with `categories: [[Classes]]`
3. **Extracts** course-code from that class
4. **Offers rename** to add course code to filename
5. **Auto-fills** class property with link to parent
